# ITMO Megabattle Backend

Единый backend экосистемы Megabattle. Он является серверным источником истины только для сайта, бота участников и его Telegram Mini App. Бот организаторов — отдельный автономный проект и не имеет доступа к этому API или базе.

## Что реализовано

- единый профиль с несколькими способами входа: Telegram, текущий Supabase Auth и будущий ITMO.ID;
- безопасное объединение identity в один профиль;
- роли участников и администраторов сайта;
- публичные профили, модерация, бан, мягкое удаление и аудит;
- контент сайта: люди, истории, партнёры и мероприятия;
- предложения историй участниками и модерация;
- NFC-метки, несколько меток на профиль, статистика сканирований и граф знакомств;
- XP, уровни, рейтинг, факультетский зачёт, внутренняя валюта, достижения и одноразовые коды;
- регистрация, очередь ожидания и отметка на мероприятиях;
- обязательный ITMO.ID для выбранных официальных действий;
- адаптер ITMO Events API и webhook;
- очередь Telegram-уведомлений с retry и idempotency;
- временные фото с автоматическим удалением;
- AES-256-GCM хранилище проектных паролей с дополнительным четырёхзначным PIN;
- ограниченный журнал последних 50 административных действий;
- запуск как обычный Node.js-сервис на VPS и serverless-адаптер для Vercel.

Подробные контракты находятся в [docs/API.md](docs/API.md), схема системы — в [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md), запуск — в [docs/DEPLOY.md](docs/DEPLOY.md).

## Быстрый локальный запуск

```bash
cp .env.example .env
npm install
npm run dev
```

API запустится на `http://localhost:4000`. Проверка:

```bash
curl http://localhost:4000/health
```

Для полноценной работы нужна база Supabase и миграции из `supabase/migrations` в порядке имён:

1. `202607010000_baseline.sql`;
2. `20260705_story_submissions.sql`;
3. `202607150001_ecosystem_core.sql`;
4. `202607150002_full_platform.sql`.

Для существующего проекта, в котором baseline уже применён вручную, выполняются только ещё не применённые миграции.

## Проверка качества


## Источник истины

PostgreSQL/Supabase хранит профили, роли, контент сайта, NFC-метки, связи, баллы и регистрации. ITMO Events является внешней системой регистрации, а backend синхронизирует её идентификаторы и статусы. Данные бота организаторов здесь не хранятся.

Сайт и Mini Apps больше не должны выполнять привилегированные запросы к таблицам напрямую. Клиенты получают сессию backend и работают через `/api/v1/*`.
